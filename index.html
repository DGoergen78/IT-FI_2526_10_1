Wichtige Git-Begriffe

Repository

Ein Repository ist ein Projektordner, der von Git verwaltet wird.

Commit

Ein Commit ist ein gespeicherter Stand des Projekts.

Stage

Staging bedeutet, dass man Änderungen für den nächsten Commit vorbereitet.

Push

Push lädt lokale Commits zu GitHub hoch.

Pull

Pull lädt Änderungen von GitHub auf den eigenen Computer herunter.

Clone

Clone erstellt eine lokale Kopie eines GitHub-Repositories.

Branch

Ein Branch ist ein eigener Entwicklungszweig innerhalb eines Projekts.

Merge

Merge führt Änderungen aus verschiedenen Branches zusammen.

Conflict

Ein Conflict entsteht, wenn Git Änderungen nicht automatisch zusammenführen kann.

Remote

Ein Remote ist ein entferntes Repository, zum Beispiel auf GitHub.
